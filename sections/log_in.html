Once VM with BOSH Director is running, point your CLI to it, configure a number of environment variables to make it easier to interact with it

$ bosh -e 10.0.0.6 alias-env gcp --ca-cert <(bosh int gcp/creds.yml --path /director_ssl/ca)

$ export BOSH_CLIENT=admin

$ export BOSH_CLIENT_SECRET=`bosh int gcp/creds.yml --path /admin_password`

$ export BOSH_ENVIRONMENT=gcp

Using environment '10.0.0.6' as anonymous user Name Bosh Lite Director UUID ... Version 261.4.0 (00000000) CPI warden_cpi Features compiled_package_cache: disabled config_server: disabled dns: disabled snapshots: disabled User (not logged in) Succeeded

Obtain generated password to BOSH Director:

$ bosh int vbox/creds.yml --path /admin_password

Log in using admin username and generated password:

$ bosh login

Now we are ready to deploy!
